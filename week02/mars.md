# 텍스트 파일과 이진(Binary) 파일의 비교

## 1. 텍스트 파일 (Text File)
**개념**
* 사람이 읽고 이해할 수 있는 문자(텍스트)들로 이루어진 파일
* 데이터가 특정 문자 인코딩(예: UTF-8, ASCII) 규칙에 따라 저장
* **대표 확장자:** `.txt`, `.csv`, `.py`, `.md` 등

**장점**
* **뛰어난 가독성:** 메모장 같은 기본 텍스트 에디터로 열어서 바로 내용을 확인하고 직관적으로 수정 가능
* **높은 호환성:** 운영체제나 시스템이 달라도 데이터를 쉽게 주고받고 열어볼 수 있음

**단점**
* **처리 속도 저하:** 컴퓨터가 파일을 읽고 쓸 때 문자를 컴퓨터의 언어(0과 1)로 변환(인코딩/디코딩)하는 과정이 추가로 필요해 속도가 상대적으로 느림
* **비효율적인 용량:** 숫자 데이터라도 문자로 취급하여 하나씩 공간을 차지하므로, 파일 크기가 상대적으로 큼

---

## 2. 이진 파일 (Binary File)
**개념**
* 컴퓨터가 직접 이해할 수 있는 `0`과 `1`의 데이터(Byte) 덩어리로만 저장된 파일
* 사람의 언어로 번역하는 과정 없이 데이터가 메모리에 있는 형태 그대로 파일에 기록
* **대표 확장자:** `.bin`, `.exe`, `.jpg`, `.mp4` 등

**장점**
* **빠른 처리 속도:** 컴퓨터가 별도의 해석(변환) 과정 없이 바로 데이터를 읽고 쓸 수 있어 처리 속도가 매우 빠름
* **효율적인 용량:** 데이터를 압축된 형태로 저장하기 때문에 저장 공간을 아낄 수 있음

**단점**
* **가독성 부족:** 일반 텍스트 에디터로 강제로 열면 내용이 깨져 알아 볼 수 없음  
* **전용 프로그램 필요:** 내용을 확인하거나 수정하려면 해당 이진 파일을 올바르게 해석할 수 있는 전용 뷰어나 프로그램이 필요

---

## 3. 핵심 요약 비교표

| 구분 | 텍스트 파일 (Text) | 이진 파일 (Binary) |
| :--- | :--- | :--- |
| **구성 요소** | 사람의 언어 (문자열) | 컴퓨터의 언어 (0과 1의 Byte) |
| **가독성** | 높음 (사람이 바로 읽고 수정 가능) | 낮음 (사람이 바로 읽을 수 없음) |
| **처리 속도** | 상대적으로 느림 (변환 과정 필요) | 매우 빠름 (변환 과정 생략) |
| **파일 용량** | 큼 | 작음 (공간 효율적) |
| **주요 사용처**| 설정 파일, 소스 코드, 소규모 데이터 | 이미지, 영상, 실행 프로그램, 대용량 데이터 |
